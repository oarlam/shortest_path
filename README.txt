Цей проєкт реалізує систему пошуку найкоротшого шляху між двома точками на карті  світу, використовуючи дані OpenStreetMap. Основна вимога — порівняння власних алгоритмів пошуку на графі.

Опис структури коду
Програма складається з трьох основних модулів:

1. load_data(place_name)
Відповідає за завантаження даних безпосередньо через бібліотеку OSMnx. Функція отримує граф дорожньої мережі для заданої локації та конвертує його у зручну структуру даних (наприклад, словник суміжності) для подальшої обробки алгоритмами.

2. find_shortest_path(graph, start, end)
Основна логіка проєкту. В цьому блоці реалізовано три різні підходи до пошуку шляху. Функція приймає граф, стартову та кінцеву точки, і повертає послідовність вузлів маршруту. Вбудовані методи пошуку NetworkX не використовуються.

3. visualize_route(graph, path)
Графічний блок. Відповідає за візуалізацію карти та побудованого маршруту. Дозволяє наочно побачити результат роботи алгоритмів.

Розподіл ролей
Кожен учасник команди відповідає за реалізацію конкретної частини:
Артем: Алгоритм Dijkstra.
Олеся: Алгоритм A*.
Злата: Алгоритм Greedy.
Михайло та Захар: Робота з даними, візуалізація, документація (README)

Вимоги до оточення
Для роботи програми необхідні наступні бібліотеки:
osmnx, networkx, matplotlib.

Запуск
Після запуску скрипт завантажує карту, виконує пошук шляху обраними алгоритмами, виводить результати та візуалізацію.
