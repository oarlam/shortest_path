## Опис проєкту
Ми створили програму, яка будує маршрути між двома точками на реальній карті світу, використовуючи дані з OpenStreetMap (OSM).

Мета проєкту - розробити програмне забезпечення для навігації на реальній карті світу та дослідити ефективність різних математичних алгоритмів (Дейкстра, A*, Greedy best-first search) при пошуку оптимального маршруту в умовах реальної дорожньої мережі.

Головна ідея проєкту — не використовувати готові функції пошуку шляху з бібліотек, а написати свої власні реалізації трьох популярних алгоритмів і порівняти, як вони працюють на реальних дорогах.

---

## Використання принципів дискретної математики

У нашому коді карта міста представлена як граф, де:
* Вершини (Nodes) — це перехрестя або тупики. У кожної вершини є свої координати x та y.
* Ребра (Edges) — це дороги, що з'єднують вершини. Оскільки на дорогах буває односторонній рух, наш граф є орієнтованим.
* Вага (Weights) — це довжина дороги в метрах.

## Реалізовані алгоритми

### 1. Алгоритм Дейкстри (Dijkstra Algorithm)
Це класичний алгоритм для пошуку найкоротшого шляху.
* Як працює: Він поступово перевіряє всі можливі шляхи від старту, обираючи найближчу невідвідану вершину. Ми використовували словник для збереження відстаней і ініціалізували їх як нескінченність.
* Результат: Цей алгоритм завжди знаходить найкоротший шлях, але працює довше за інші, бо перевіряє дуже багато вершин навколо.

### 2. Greedy best-first search
Цей алгоритм намагається вгадати шлях, обираючи найкращий варіант "тут і зараз".
* Як працює: На кожному перехресті він дивиться на сусідів і обирає того, який фізично найближче до фінішу по прямій лінії. Ми рахували це за формулою відстані між координатами.
* Результат: Працює дуже швидко, але часто помиляється. Він може завести в глухий кут або побудувати не найкоротший маршрут, бо не знає про перешкоди попереду.

### 3. Алгоритм A* (A-Star)
Це вдосконалений алгоритм, який поєднує логіку Дейкстри і Greedy best-first search.
* Як працює: Він враховує і ту відстань, яку ми вже пройшли від старту (g), і приблизну відстань до фінішу по прямій (h). Сума цих значень (f = g + h) допомагає обирати найперспективніші вершини.
* Результат: Це найкращий варіант. Він знаходить найкоротший шлях, як Дейкстра, але робить це набагато швидше, бо рухається цілеспрямовано до фінішу.

## Використання

Де і як це можна використати:
Цей проєкт показує, як працюють навігатори зсередини (наприклад, Google Maps). Він корисний для:
* Розуміння навігації: Допомагає зрозуміти, як програма обирає дорогу для таксі чи пішохода.
* Планування доставки: Такі алгоритми використовують пошти та служби доставки, щоб економити пальне.
* Навчання: Наша програма наочно показує різницю між алгоритмами (де один "зрізає" шлях, а інший йде в обхід), що корисно для студентів.

## Технічна частина

Ми використовували такі бібліотеки:
* OSMnx — щоб скачувати карту міста і перетворювати її в граф.
* NetworkX — використовували тільки як структуру для зберігання графа, вбудовані алгоритми пошуку не використовували.
* Matplotlib — для малювання карти і маршрутів.
* os
* sys
* argparse

Структура коду:
1. Спочатку функція load_data завантажує карту і робить з неї зручний словник.
2. Потім функції алгоритмів шукають шлях по цьому словнику.
3. В кінці функція plot_multiple_paths малює результат, щоб ми могли порівняти маршрути візуально.

## Як запустити

Запустіть програму завдяки argparse:

python main.py --start "Kyiv, Ukraine" --end "Lviv, Ukraine" --algo dijkstra

Доступні аргументи:

--file: Path to file from which we load map

--source: ID of source node from which we start the search

--target: ID of target node
